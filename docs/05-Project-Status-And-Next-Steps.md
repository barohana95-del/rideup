# RideUp — סטטוס פרויקט ותוכנית עבודה מקיפה (Status & Action Plan)

**תאריך עדכון:** מאי 2026

מסמך זה מרכז את כל מה שבוצע עד כה בפרויקט, ואת כל מה שנותר לעשות עד להשקה המלאה של המערכת, כולל כל פרט מקצה לקצה.

---

## ✅ מה בוצע עד כה (הושלם)

### 1. תשתית וניתוב (Multi-tenant)
- הקמת סביבת פיתוח (Vite + React).
- יצירת מערכת ניתוב חכמה (Router) שיודעת להפריד בין: דף נחיתה ראשי, פאנל ניהול (Admin), ואתר הלקוח עצמו (Public RSVP).

### 2. הקמת לקוח (Onboarding Wizard)
- פיתוח מערכת רישום רב-שלבית (Wizard) יוקרתית.
- בחירת חבילה, קביעת כתובת האתר (Slug), בחירת שפת עיצוב (Theme), פרטי אירוע, הוספת ערים, משמרות, טקסטים, וסיום מוצלח.

### 3. עמוד נחיתה ותדמית (Marketing Site)
- פיתוח עמוד נחיתה בסטנדרט גבוה מאוד תחת קונספט ה-"Industrial Luxury".
- אזור Hero דינמי עם אוטובוס נכנס משמאל ואפקטים עדינים של תאורה ברקע.
- מערכת רספונסיבית, אנימציות גלילה (Framer Motion).
- עיצוב אלגנטי לסקשנים: אודות, פיצ'רים, מחירון, סימולטור הסעות, שאלות נפוצות, ספקים ו-Footer נגיש ומותאם SEO.

### 4. אתר אישורי ההגעה (Public RSVP)
- טופס הרשמה דינמי המותאם למובייל להזנת פרטי אורחים, כמות מלווים, בחירת משמרת נסיעה ועיר איסוף.
- עיצוב "Classic" אסתטי.

### 5. בסיס לפאנל ניהול (Admin Panel Shell)
- יצירת תשתית ממשק הניהול של מארגן האירוע.
- לשונית "נרשמים" עם תצוגת טבלה לאישורי ההגעה.

---

## 🚀 מה נשאר לעשות (Roadmap להשלמת המערכת)

השלבים הבאים מסודרים לפי סדר הגיוני של פיתוח מקצה-לקצה (Vertical Slices):

### שלב 1: עורך העיצוב בפאנל הניהול (Design Editor)
*המטרה: לאפשר לבעל האירוע לשנות את מראה אתר ההרשמה בעצמו.*
- [ ] פיתוח לשונית "עיצוב" בפאנל הניהול.
- [ ] בחירת צבע מותאם אישית (Primary Color Picker).
- [ ] כלי להעלאת תמונת כיסוי (Cover Image) ולוגו לאירוע.
- [ ] יצירת Preview דינמי בזמן אמת של איך האתר ייראה.
- [ ] עדכון שרת (API) לשמירת נתוני העיצוב בטבלת `tenant_settings`.
- [ ] יישום ההגדרות האלו בטופס אישורי ההגעה בפועל (Public RSVP).

### שלב 2: תבניות עיצוב (Themes System)
*המטרה: מערכת מולטי-דיזיין לבחירת קונספט האירוע.*
- [ ] פיתוח תבנית **Modern** (עיצוב מודרני, נקי וטכנולוגי).
- [ ] פיתוח תבנית **Rustic** (טבע, גווני אדמה, פונט כפרי).
- [ ] פיתוח תבנית **Festive** (מסיבה, זהב, יוקרה).
- [ ] הוספת בחירת התבנית ללשונית העיצוב בפאנל הניהול.

### שלב 3: ליבת הלוגיסטיקה - אלגוריתם ההסעות והפקת דוחות
*המטרה: הערך המוסף המרכזי של המערכת.*
- [ ] יצירת עמוד "תכנון נסיעות" בפאנל הניהול.
- [ ] פיתוח אלגוריתם (First-Fit-Decreasing) שמחלק אנשים ממשמרות וערים לאוטובוסים לפי התפוסה (לדוגמה אוטובוס של 50, מיניבוס של 20).
- [ ] הצגה ויזואלית של האוטובוסים ומספר המושבים הפנויים בכל אחד.
- [ ] אפשרות ייצוא לאקסל (Excel Export) של רשימת הנוסעים עבור נהג האוטובוס.

### שלב 4: מודל תשלומים ומנויים (Billing & SaaS)
*המטרה: להפוך את המערכת למוצר שמכניס כסף.*
- [ ] חיבור חברת סליקה (PayPlus / Stripe / Tranzila).
- [ ] פיתוח מסך Checkout לאימות וסליקת כרטיסי אשראי.
- [ ] יצירת Webhook בצד שרת שמקבל את אישור התשלום ומעביר את האירוע מסטטוס `trial` לסטטוס `paid`.
- [ ] הנפקת חשבונית מס אוטומטית ללקוח ושליחתה במייל.

### שלב 5: התראות, תכתובות ואימיילים (Email Notifications)
*המטרה: קשר רציף מול הלקוח ואישורים לאורחים.*
- [ ] חיבור לשירות שליחת מיילים (לדוגמה: Brevo / SendGrid).
- [ ] מייל אוטומטי לאורח לאחר השלמת הרשמה ("ההרשמה שלך להסעה לאירוע אושרה").
- [ ] מייל עדכון יומי / שבועי למארגן ("יש לך 15 נרשמים חדשים להסעות").

### שלב 6: מחזור חיי האירוע (Lifecycle & Archiving)
*המטרה: התנהלות אוטומטית שחוסכת תחזוקה ידנית.*
- [ ] פיתוח תהליך רקע (Cron Job) הסורק תאריכי אירועים פגי תוקף.
- [ ] נעילת טופס ההרשמה של אירוע שעבר (או שפג לו מנוי ה-Trial).
- [ ] העברת אירועים לסטטוס `Archived` - צפייה לקריאה בלבד בפאנל (Read-only mode).
- [ ] התראות במייל ללקוח: "נשארו 3 ימים לסיום תקופת הניסיון שלך".

### שלב 7: דאשבורד וסטטיסטיקה מורחבת (Analytics)
*המטרה: שליטה במספרים.*
- [ ] יצירת דאשבורד יפהפה בפאנל הניהול למארגן האירוע.
- [ ] תצוגת פאי / גרפים המראים את התפלגות הנרשמים לפי עיר ולפי משמרת חזרה.
- [ ] צפי משוער של עלויות הסעה בהתאם לכמות האוטובוסים הנדרשת.

### שלב 8: הכנות פינליות להשקה (Launch & QA)
*המטרה: יציאה לאוויר בצורה חלקה.*
- [ ] העלאה לשרת ייצור אמיתי (Production Environment) - חיבור דומיין אמיתי ותעודת SSL Wildcard שמאפשרת קליטת סאב-דומיינים (`*.rideup.co.il`).
- [ ] בחינת חדירות בסיסית ואבטחת מידע (הגנה מפני SQL Injection, XSS, CSRF).
- [ ] רישום למסוף גוגל (Google Analytics & Search Console).
- [ ] הטמעת מערכת עזרה (Docs / FAQ פנימי בפאנל הלקוח).
- [ ] כניסה לתקופת "Soft Launch" עם מספר לקוחות בטא נבחרים לפני שיווק המוני.

---

**סיכום:** המערכת שלנו יציבה, חזקה, ונראית מדהים מבחינת קו עיצובי. המנוע הבסיסי כבר חי ובועט. המטרה כעת היא להתחיל לצמצם את "שלב 1" ולהביא את האפליקציה למצב בו מארגן אירוע מתאים אישית את האתר שלו.
