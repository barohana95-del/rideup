# RideUp — תוכנית עבודה להמשך (Work Plan)

**עדכון אחרון:** מאי 2026
**מטרה:** המשך פיתוח יעיל וממוקד לאחר השלמת התשתית המרכזית ואתר אישורי ההגעה.

---

## 1. סטטוס פיתוח נוכחי
- **[הושלם]** תשתית המערכת וניתוב (Multi-tenant Router).
- **[הושלם]** מערכת הרשמה והקמת לקוח (Onboarding Wizard) שפועלת מקצה לקצה.
- **[הושלם]** שלב 4 (RSVP): בניית מנוע צד השרת להצגה וקליטת אישורי הגעה דינמיים (`config.php`, `register.php`), כולל ממשק "Classic Theme" אסתטי.
- **[הושלם חלקית]** פאנל ניהול (Admin Panel): צפייה בנרשמים וסטטיסטיקות עובד, אך חסרה עדיין יכולת שליטה על העיצוב.

---

## 2. יעדים לביצוע — לפי סדר עדיפויות

### יעד A: עורך העיצוב בפאנל הניהול (Editor Mode)
*מתן שליטה למארגן האירוע על המראה של אתר ה-RSVP.*
- [ ] יצירת לשונית "עיצוב" תחת ה-Admin Panel.
- [ ] הוספת Color Picker כך שהלקוח יוכל לבחור את הצבע הראשי (`primaryColor`).
- [ ] העלאת תמונת רקע (Cover Image) ולוגו מותאם.
- [ ] חיבור צד שרת לשמירת הנתונים הללו בתוך טבלת `tenant_settings`.

### יעד B: דף נחיתה תדמיתי (Marketing Landing Page - שלב 6)
*בניית "חלון הראווה" של RideUp שישכנע וימשוך לקוחות.*
- [ ] עיצוב אזור ה-Hero (פתיח) דינמי ויוקרתי (בסגנון Industrial Luxury).
- [ ] הסבר קצר על המוצר "איך זה עובד ב-3 צעדים".
- [ ] אזור Pricing (חבילות ומחירים) המציג Trial, Basic, Pro.
- [ ] יצירת מעבר חלק מהנחיתה אל תוך ה-Onboarding.

### יעד C: אינטגרציית תשלומים (Billing - שלב 7)
*מעבר לשלב העסקי — גביית תשלום ופתיחת החסימה מה-Tenant.*
- [ ] חיבור לספק תשלומים (PayPlus / Tranzila / Stripe Sandbox).
- [ ] בניית מסך תשלום מאובטח שמקשר בין הפרופיל לקבלת האישור.
- [ ] פיתוח Webhook שיודע לעדכן את הסטטוס מ-`trial` ל-`active` ברגע שהתשלום הצליח.

### יעד D: גיוון עיצובי (Themes - שלב 8)
*הוספת סגנונות עיצוב שיאפשרו התאמה מושלמת לכל אירוע.*
- [ ] עיצוב ופיתוח `ModernTheme` — קווים נקיים, פונטים מודרניים.
- [ ] עיצוב ופיתוח `RusticTheme` — צבעים אדמתיים וחמימים (טבע/כפר).
- [ ] עיצוב ופיתוח `FestiveTheme` — זהב, מסיבה ויוקרה.
- [ ] חיבור המעבר בין התבניות ללשונית ה"עיצוב" בפאנל הניהול.

### יעד E: ניהול מחזור חיים וארכיון (Lifecycle - שלב 9)
*סגירת מעגל: מה קורה כשהאירוע נגמר או אם התשלום פג.*
- [ ] בניית מנגנון (Cron / בדיקת זמנים) המעביר אירוע שהסתיים לסטטוס `archived`.
- [ ] חסימת אפשרות ההרשמה באתרים של אירועים שעברו או אתרים שפג להם ה-Trial.
- [ ] הוספת "מצב קריאה בלבד" למשתמש שרוצה להיכנס לייצא אקסלים ישנים.

## 3. כללי אצבע להמשך הפיתוח
1. **Vertical Slices:** לא מפתחים משהו "באוויר". כל פיצ'ר מקבל טיפול מקצה לקצה (Frontend -> API -> Database) עד שהוא עובד וניתן לבדיקה.
2. **Industrial Luxury:** נראות וגימור חשובים. אנחנו מקפידים על ריווח אדריכלי, טיפוגרפיה איכותית והתאמה מושלמת למסכים ניידים (Responsive).
3. **שמירת יציבות:** הקוד נשמר מודולרי. מערכת ה-Routing וה-Tenant Detection כבר עובדת ותשמש אותנו כעוגן.

**הצעד הבא ליישום בפועל:** לפי בחירתך, נתחיל עם **יעד A (עורך העיצוב)** או עם **יעד B (דף הנחיתה)**.
